# Issue 处理记录（按时间倒序）

1. **物品状态与附件管理联动优化** — 重构状态修改入口，完善上传回显、购买记录附件查看、字典快捷新建与质保自动计算。
2. **统一附件管理后端落地** — 新增 `file_attachment` 表、附件上传/查询/删除接口与 URL 拼接工具，提供统一对象存储访问能力。
3. **后端缺失统一错误码处理** — 已新增 `ErrorCode` 与 `BizException`，统一 `{code,data,msg}` 返回。
4. **资产列表指标不符设计** — 重写 `AssetServiceImpl`，输出总投入、日均成本、使用天数。
5. **缺少 MinIO 文件上传能力** — 新增 `FileController`、MinIO 客户端与配置。
6. **资产售出缺乏向导接口** — 实现 `/api/assets/{id}/sell`，联动状态与净收入。
7. **心愿单无 CRUD 服务** — 补全 `WishlistService` 与 REST 接口，支持转资产。
8. **数据库结构与设计文档不一致** — 更新 schema 与种子数据，引入中文状态、心愿单表。
9. **前端无法管理资产** — 新增资产列表、详情、弹窗表单，支持上传与出售。
10. **心愿单前端缺失** — 新增心愿单页面，支持新增、编辑、删除、转资产。
11. **仪表盘未体现关键指标** — 重新设计 Dashboard，展示总投入、日均成本、快速出售入口。
12. **缺失接口文档** — 编写 `deploy/openapi.yaml`，覆盖上传、资产与心愿单接口。
13. **字典与物品文档滞后** — 更新 README、设计文档及 Postman 集合以匹配 V0.3.3 新增的类别/平台/标签与物品字段。
